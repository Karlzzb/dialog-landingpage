# ADR 0003：结构化会话记忆 + 自包含查询改写

状态：已接受
日期：2026-07-09

## 背景

E2E-04 要求多轮上下文继承：第2轮"江夏区有额外的吗？"须继承第1轮的"武汉市/校企合作"。
bisheng 版本常在此翻车（隐含限定词丢失）。
全量对话历史入 prompt 既 token 爆炸，又不能可靠解决隐含指代——模型仍可能忽略。

## 决策

- **结构化会话记忆**替代全量历史，追求 context 最小。
只维护：用户画像、权限范围、滚动的"关键实体/约束摘要"。
- 每轮进内核前有一个**改写步**：基于会话记忆把用户输入补全成**自包含查询**，将隐含约束变为显式查询实体。
- 改写步是入口确定性节点，但**只做指代消解、不做流程分叉**，因此与 ADR 0001「不设入口硬分类器」不冲突。

## 备选方案

- **全量对话历史入 prompt**：token 爆炸且隐含指代易漏——被否。
- **全量历史 + 结构化摘要混合**：最稳但状态最重，当前不需要——暂不采用。

## 影响

- E2E-04 的失败模式（隐含限定词丢失）被显式改写步结构化解决，可对上其 Pass 标准。
- 会话记忆天然承载权限/画像等会话级不变量（T1：轮次>1 不重复初始化）。
