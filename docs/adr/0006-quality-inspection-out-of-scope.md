# ADR 0006：质检体系独立于对话 agent，不在本设计范围

状态：已接受
日期：2026-07-09

## 背景

`design.md` 第二部分定义了约 25 条质检项（T1-T28，分确定性/判断/行为三类，各有通过门槛与打分规则）。
一个诱人的错误是把它们做成 agent 图里的同步校验节点——让 agent 边干活边给自己打分。
这会加延迟、"用 LLM 判 LLM"不可靠、且混淆了"能力"与"评估"两件本质不同的事。

## 决策

- **质检体系（T1-T28 的打分、通过门槛、E2E、离线标注）是独立系统**，把本 agent 当黑盒来评分/审计。
不属于本 agent 设计，本期不设计它。将来可作为独立能力/服务存在。
- **区分固有安全属性 vs 质检**：只读、行级权限、PII 脱敏不是"质检"，而是数据访问层的固有性质，已由 ADR 0004（参数化查询能力）+ ADR 0005（权限注入层）结构性保证。
Agent 从设计上就只可能拿到有权、已脱敏的数据，无需事后"脱敏质检门"补救。
- T16-T20 描述的内外部数据隔离、盲区坦诚、结论覆盖，是**结论生成这一步的输出行为**（靠 prompt + 结构化素材达成），不是独立质检关卡。

## 影响

- **删除**原设想的"出口同步质检门"节点。
- 安全 = 数据层固有性质；输出质量 = 结论生成本身；打分/审计 = 外部独立系统。
- agent 图更薄、关键路径延迟更低、职责更单一。
