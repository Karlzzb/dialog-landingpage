# ADR 0007：三工具统一的结构化 Evidence 契约

状态：已接受
日期：2026-07-09

## 背景

结论生成要做到内外部数据隔离、来源标注、数字忠于原始数据（design.md T18/T25 的输出诉求），
前提是内核拿到的检索结果必须结构化、带来源、可回溯——而不是三个工具各返回一坨文本。

## 决策

知识库、数据库、联网三个检索工具统一返回 **Evidence**：

```
Evidence = {
  source_type: INTERNAL_KNOWLEDGE | INTERNAL_DATABASE | EXTERNAL_SEARCH,
  content:  原始片段 / 字段值,
  citation: 文件名 / 条款号 / URL / 表字段,   // 可回溯
  raw:      原始 JSON（数据库工具尤其重要，供数字溯源）
}
```

- 结论生成末步据此天然区分内外部、自动加"据互联网公开信息"、数字直接引 raw 字段。
- 输出质量行为由数据契约导出，而非事后质检。
- **联网内容安全过滤**内置在联网工具边界：外部内容进 state 前清洗/拦截，绝不裸奔进 LLM。

## 影响

- 所有工具与结论生成 prompt 都依赖此契约，是输出质量的地基。
- 新增检索工具须遵守同一契约。
