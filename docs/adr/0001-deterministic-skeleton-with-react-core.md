# ADR 0001：确定性骨架 + ReAct 内核

状态：已接受
日期：2026-07-09

## 背景

产教融合专家助理原为 bisheng 实现，用 53 个节点、大量 condition 硬接成一条流水线（流程式）。
问题：不具备真实智能体能力，无法自主多步推理与工具编排。
但 `design.md` 又规定了大量"确定性条目"（权限边界、PII 脱敏、注入防护、渐进查询命中即停、数字反向溯源），要求 100% 通过、可一票否决。
这两者存在张力：纯 ReAct loop 智能但护栏难保证；纯流水线可控但不智能。

## 决策

采用 **确定性骨架 + ReAct 内核** 的分层架构，技术栈 LangGraph：

- 薄的确定性外层图：入口做初始化/画像/权限/意图分流，出口做质检门。
确定性条目落在这些边界节点上强制执行，不依赖 LLM 自觉。
- 中间是真正的 ReAct 循环（agent 节点 + conditional loop 回边），负责规划、调工具、观察、再决策。

## 备选方案

- **纯单体 ReAct Agent**：最智能，但确定性护栏最难保证——被否。
- **多智能体 Supervisor**：表达力最强，但对当前单领域助理过度设计，调试与确定性保证更复杂——暂不采用，留待"对话/检索/分析"需独立演进时再引入。

## 影响

- 护栏是图的硬约束（结构层面保证），推理是 agent 的自由。
- 未来若需拆分子智能体，可在内核处替换为 supervisor，骨架不变。
